Discovery

Customer Need

I seek to understand the current process and the intended outcome through basic questions and interaction with users.

Once the outcome is agreed upon, I make sure there is clarify the plan to ensure the scope of the project is defined.

Discovery

Documentation

Once the customer need is defined, then documentation is composed to ensure that project can be tracked properly.

User stories are defined and tasks are entered into the Project Management app to establish timelines.

Discovery

Design

Mockups are created in Figma to ensure that the design is agreed upon. This includes determining page navigation and required menus.

Once the prototype is finished, it is once again reviewed with the customer to ensure the intended outcome is agreed upon one last time.

Development

Architecture

The project's initial layout is done using DRY principles and intended to use the most efficient processing methods possible.

Prior to coding, the layout is to align with the tasks listed in the Project Management database.

Development

Programming

Coding is done piece by piece starting with the initial HTML to create the content. This is followed by the necessary CSS and Javascript.

Once the content is created, every change possible is submitted to GitHub.

Development

Unit Testing

As each section of code is completed, it is tested for functionality to ensure that the program is working properly.

It is also tested for aesthetics and to ensure that links work properly.

Support

Troubleshooting

Any bugs found in the program will be handled based on customer need and tracked in the Project Management system.

Updates and corrections will be noted in GitHub as new versions of the code are submitted.

Support

Enhancement

Enhancement ideas restarts the process. These will be done sprints and tested accordingly.

Once the updates are authorized by the customer, the new version will be deployed.
